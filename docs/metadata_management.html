MetaData Management 
Metadata Management is becoming more important ...
There is continued pressure and interest in being able to provide access to an integrated view of corporate in formation.

This is being encouraged by CRM, which requires one single, consistent view of all sources of Customer data.
There are different approaches to the design of a Metadata Architecture.
Here's one that represents a good starting-point :-

  1. Define the requirements - eg provide different views to the same enterprise MetaData.
  2. Design the Data Model for the database which will provide the Repository.
  3. Carry out a Walkthrough using a small sample of actual metadata with the Users.
  4. Review the available commercial products to meet the requirements.

There is also a Page discussing MetaData Repositories, and a Data Model for MetaData Management.
