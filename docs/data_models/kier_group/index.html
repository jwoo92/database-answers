Data Model for the Kier Group   
We have no connection whatsoever with the Kier Group.
We created this Data Model simply because it looked like an interesting challenge.
And we are glad to say that is exactly how it turned out.
Then we went to Westfield and realised that we could apply the same approach to the Capita Group.
An Access Database is available on demand.
The Specifications for the Data Model were provided in a question to our Web Site.

We might design a Data Mart, which is very useful for providng data for Reports and Business Intelligence.

We would be glad to have your comments.

If you are new to Data Models, this page of my new Tutorial will help you understand the Data Model.


Step 1 : First we define the 'Things of Interest'
which are a Project Platform, Work Orders and three Sectors - Defense, Education and Other Sectors.

On the Kier Group Web Site, we find a reference to the importance of Supply Chain management, so we decide to add it to the Project Platform.


Step 2: Then we think about how they might look in a Conceptual Data Model.




Step 3: Here we add the details required to create the Logical (ERD) Data Model
which are the Key fields and basic Attributes.
This is how they might look in a 3NF Normalised Data Model.
