Back to the Bookings Data Model. 
The purpose of the Database is to
support taking Bookings for Events, such as Wedding Receptions.

The User Requirements have been defined by Michael as follows :-

"I input the credentials of the calling person inquiring about a function.

Later on they choose a date, pay down a deposit to secure a date, then they choose a menu.

At the same time they tell us of many changes that need to be noted.

The whole idea is to collect all the Booking Notes in the process so that they can be printed and given to the Banquet Manager for execution."


Things of Interest :-
1) Bookings
2) Booking Notes
3) Customers
4) Menus
5) Menu Changes
