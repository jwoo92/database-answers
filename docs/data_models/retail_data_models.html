  Retail Data Models
 
These Retail Data Models all follow a Design Pattern which has a Top-Level Model with separate Models for specific Subject Areas.
Click here for a Data Architecture showing how a Retail Data Model can be created from a Canonical Data Model and Subject Area Models.

We have three options when it comes to designing a Retail Enterprise Data Model :-
1) Top-down - starting with a 'Mission Statement'
2) Bottom-up - starting with original source documents, such as Financial Reports, Invoices, Sales Receipts and so on
3) 'Middle-out' - starting with a Life Cycle analysis for Customers, Events or Products
